5.1 快速開啟報告

雙擊專案根目錄的 開啟測試報告.bat，會以系統瀏覽器開啟最新的 TestResultReport.html。

建議優先閱讀 TestResultReport.html（步驟級 HTML 報告），含每步操作截圖，最適合除錯。

5.2 報告檔案位置

報告類型主要路徑用途
TestResultReport .../bin/Release/net8.0-windows/reports/TestResultReport.html 步驟級 Pass/Fail、每步截圖、錯誤訊息（主要報告）
ExtentReports .../bin/Release/net8.0-windows/reports/SemiInspectionTestReport.html 傳統 HTML 報告、stack trace
同步副本（Web 控制台用） .../Testcase_demo2_desktop_FlaUI_BDD/reports/ Web 6690 讀取此目錄
別名 Automation_testcase/Project_FlaUIBDD/Testcase_demo2_desktop_FlaUI_BDD/reports/Demo2TestReport.html Web 控制台別名 → SemiInspectionTestReport（測試後產生）
JUnit XML reports/junit-results.xml CI 整合、Web 控制台解析
失敗截圖 bin/Release/net8.0-windows/Screenshots/ 點擊/步驟/Scenario 結束截圖

完整路徑範例（相對 SQA_Inspetion_App）：

Automation_testcase/Project_FlaUIBDD/Testcase_demo2_desktop_FlaUI_BDD/ bin/Release/net8.0-windows/reports/TestResultReport.html reports/TestResultReport.html ← Web 控制台同步副本

5.3 TestResultReport.html 結構

頁首（Report Header）

  • Feature 名稱：Semi Inspection Desktop 測試
  • 執行時間區間
  • Feature 描述（User Story）

Summary 摘要列

欄位說明
Test Cases執行的 TC 總數
Passed通過數（綠色）
Failed失敗數（紅色）
Duration總耗時

每個 Test Case 區塊

點擊 TC 標題列可展開/收合，內容包含：

  • TC 編號與標題 — 如 TC01 - Import Recipe 至 Recipe_data
  • 狀態徽章 — PASSED（綠）/ FAILED（紅）
  • 標籤 — @Functional @Import 等
  • 錯誤訊息 — 失敗時顯示 Assert 或 Exception 內容
  • 步驟表 — 每個 Given/When/Then 的 Pass/Fail、耗時
  • 操作截圖 — 每步的 UI 截圖，點擊可 lightbox 放大

5.4 如何解讀失敗報告

  1. 在 Summary 確認 Failed 數量
  2. 找到紅色邊框的 TC 區塊（testcase fail）
  3. 閱讀 tc-error 區塊的錯誤訊息
  4. 在步驟表中找到第一個 FAIL 的步驟
  5. 查看該步驟的操作截圖，確認 UI 當時狀態
  6. 對照 第 4 章 進行修復
提示：若 TestResultReport 截圖路徑無法顯示，直接到 Screenshots/ 目錄找同時間戳的 PNG 檔。

5.5 SemiInspectionTestReport.html（ExtentReports）

傳統 ExtentReports 格式，適合查看：

  • 完整 Exception stack trace
  • Scenario 級別的 Pass/Fail 時間軸
  • 嵌入的失敗截圖

路徑：Automation_testcase/Project_FlaUIBDD/Testcase_demo2_desktop_FlaUI_BDD/reports/SemiInspectionTestReport.html（Web 控制台同步副本）

5.6 JUnit XML（junit-results.xml）

供 CI/CD 或 Web 控制台解析，結構範例：

<testcase name="TC01 - Import Recipe 至 Recipe_data" time="25.3"> <failure message="Expected log to contain 'Import Recipe'"> ... stack trace ... </failure> </testcase>

Web 控制台「測試結果」分頁會解析此檔顯示通過率與失敗清單。

5.7 三種開啟報告的方式

方式操作適用情境
批次檔 雙擊 開啟測試報告.bat CLI 跑完後快速查看
直接開檔 瀏覽器開啟 bin/Release/.../reports/TestResultReport.html 開發除錯
Web 控制台 啟動測試平台.bat → 測試結果分頁 → 點報告連結 勾選 TC 執行後線上查看

5.8 報告產生設定（App.config）

鍵預設效果
TakeScreenshotOnClicktrue每次 UI 操作截圖（出現在 TestResultReport）
TakeScreenshotOnFailuretrueAssert 失敗時額外截圖
TakeScreenshotOnStepCompletetrue每 Gherkin 步驟完成截圖

報告由 Automation_testcase/Project_FlaUIBDD/Testcase_demo2_desktop_FlaUI_BDD/Helpers/TestResultReportBuilder.cs 與 HtmlReportHelper.cs 在 Hooks/TestHooks.cs 的 AfterTestRun 中產生。
