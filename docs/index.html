執行流程

以下為整個 BDD 自動化執行流程（7 步驟）：從讀取 TPS 規格、對照與實作程式碼，到建置、執行測試與確認報告。

  1. 讀取 TPS — 在 Automation_testcase/Test_cases/TPS.md 找到對應 TC 的 Given / When / Then。詳見 第 1 章
  2. 對照 Feature — 確認 .../Features/Inspection_App.feature 中 Scenario 標題、標籤、步驟語意一致。
  3. 實作 StepDefinitions — 在 .../StepDefinitions/Inspection_AppSteps.cs 新增或修改步驟綁定（只做編排與 Assert）。
  4. 實作 PageObjects — FlaUI 操作放在 Page Object，不要寫在 Steps 內。詳見 第 2 章
  5. 設定路徑 — 更新 App.config 相對路徑，或由 setup_env.bat 環境變數覆寫。
  6. 驗證 — 執行 build_semi.bat → dotnet build -c Release → 單一 TC 測試（執行單一測試.bat）。詳見 第 3 章
  7. 確認報告 — 檢查 .../reports/TestResultReport.html 已產生。詳見 第 5 章；失敗時見 第 4 章

執行 BDD Automation Testcase

執行用批次檔（.bat）

以下批次檔位於專案根目錄 SQA_Inspetion_App\，點擊名稱即可執行（需透過 開啟操作手冊.bat 開啟本頁）：

分類批次檔說明
操作手冊
手冊 開啟操作手冊.bat 啟動本機伺服器（port 6688）並開啟本手冊
docs/index.html
被測 App（Semi Inspection Desktop）
啟動 啟動InspectionApp.bat 建議入口。必要時先建置，再啟動 Semi Inspection Desktop。 內部呼叫 run_semi.bat 啟動 EXE
啟動 run_semi.bat 僅啟動被測 App（不含測試）；若 EXE 不存在會自動建置
建置 build_semi.bat 僅建置 SemiInspectionDesktop.exe，不啟動程式
測試執行
全套 run_tests.bat 建置被測 App + BDD 專案，執行全部 10 個 TC
單一 執行單一測試.bat 只跑指定 TC（TC01–TC10）；點擊後會提示輸入編號
報告與控制台
報告 開啟測試報告.bat 開啟最新 TestResultReport.html
Web 啟動測試平台.bat 啟動 Web 控制台 http://localhost:6690/
環境設定
環境 setup_env.bat 設定 APP_ROOT、ApplicationPath 等環境變數；由其他 .bat 自動呼叫，通常無需手動執行
