1.1 什麼是 TPS？

TPS（Test Procedure Specification） 在本專案中以 Gherkin 格式撰寫，存放於：

Automation_testcase/Test_cases/TPS.md

每個 TC 以 Scenario: TCxx - 簡短描述 定義 Given / When / Then 步驟，使用繁體中文。

1.2 三層文件鏈（必須同步）

新增或修改測試案例時，需依序維護以下四層文件：

順序文件用途
1Automation_testcase/Test_cases/TEST_PLAN.md測試目的、人工步驟、預期結果、Defect#
2Automation_testcase/Test_cases/SemiInspection_10_TestCases.md案例總表 + UI AutomationId 對照
3Automation_testcase/Test_cases/TPS.mdGherkin 程序規格（Given/When/Then）
—Automation_testcase/TPS_BDD_skill.mdAgent 轉換／維護 BDD 的 skill 指引
4Feature + Steps + PageObjects可執行的 BDD 自動化程式碼
規則：一個 Scenario = 一個 TC；標題格式 TCxx - 簡短描述；標籤使用 @Functional / @Negative + 領域標籤（如 @Import、@RawData）。

1.3 轉換流程（7 步驟）

  1. 讀取 TPS — 在 Automation_testcase/Test_cases/TPS.md 找到對應 TC 的 Given / When / Then。
  2. 對照 Feature — 確認 .../Features/Demo2Desktop.feature 中 Scenario 標題、標籤、步驟語意一致。
  3. 實作 StepDefinitions — 在 .../StepDefinitions/Demo2DesktopSteps.cs 新增或修改步驟綁定（只做編排與 Assert）。
  4. 實作 PageObjects — FlaUI 操作放在 Page Object，不要寫在 Steps 內。
  5. 設定路徑 — 更新 App.config 相對路徑，或由 setup_env.bat 環境變數覆寫。
  6. 驗證 — 執行 build_semi.bat → dotnet build -c Release → 單一 TC 測試。
  7. 確認報告 — 檢查 .../bin/Release/net8.0-windows/reports/TestResultReport.html 已產生。

1.4 TPS → Feature 範例（TC01）

TPS.md 中的 Scenario

@Functional @Import Scenario: TC01 - Import Recipe 至 Recipe_data Given 測試資料已就緒 And 應用程式已重新啟動 When 我點擊工具列「Import Recipe」 And 我在檔案對話框選擇樣本 InspectionRecipe_Sample.json Then Recipe_data 應存在 InspectionRecipe_Sample.json And 資料表應可見 And 日誌區應包含「Import Recipe」

對應 Feature 檔

Automation_testcase/Project_FlaUIBDD/Testcase_demo2_desktop_FlaUI_BDD/ Features/Demo2Desktop.feature

TPS 中的每個 Scenario 直接複製到 Feature 檔，步驟文字必須與 StepDefinitions 的 regex 綁定完全一致。

1.5 StepDefinitions 與 PageObject 分工

層級檔案職責
StepDefinitionsDemo2DesktopSteps.cs[Given] / [When] / [Then] 綁定；呼叫 PageObject；NUnit Assert
PageObjectMainWindowPage.cs 等FlaUI 找控制項、點擊、輸入、讀取日誌
HooksTestHooks.cs啟動/關閉 App、報告初始化、截圖
HelpersConfigHelper.cs 等路徑解析、截圖、報告產生

Step 綁定範例

[When(@"我點擊工具列「(.*)」")] public void WhenClickToolbar(string buttonName) { _mainWindowPage.ClickToolbar(buttonName); } [Then(@"日誌區應包含「(.*)」")] public void ThenLogContains(string keyword) { ClassicAssert.IsTrue(_workspacePage.LogContains(keyword)); }

1.8 完成檢查清單

  • TPS.md、Feature、Steps 步驟文字一致
  • 新步驟已在 Demo2DesktopSteps.cs 綁定
  • UI 操作在 PageObject 而非 Steps
  • App.config 路徑正確（相對 APP_ROOT）
  • dotnet test -c Release --filter "Name~TCxx" 通過
  • TestResultReport.html 含步驟截圖
