2.1 修改既有 TC 的標準流程

  1. 修改 Automation_testcase/Test_cases/TEST_PLAN.md（目的、步驟、預期）
  2. 同步 SemiInspection_10_TestCases.md（總表、AutomationId）
  3. 更新 TPS.md 的 Gherkin 步驟
  4. 同步 Features/Demo2Desktop.feature 的 Scenario
  5. 若步驟語意變更 → 修改 Demo2DesktopSteps.cs
  6. 若涉及 UI 操作 → 修改 PageObject（非 Steps）
  7. 單一 TC 驗證：執行單一測試.bat TC01
重要：永遠先改 TPS 定稿，再改 Feature 與程式碼。避免 Feature 與 TPS 不一致。

2.2 可修改的檔案與用途

檔案何時修改範例
Demo2Desktop.feature步驟文字、標籤、Scenario 標題新增 Then 斷言步驟
Demo2DesktopSteps.cs新 Gherkin 步驟需綁定 C# 方法[When(@"我在檔案樹雙擊 (.*)")]
MainWindowPage.cs工具列按鈕、快捷鍵、About新增 ToolbarMap 項目
WorkspacePage.cs檔案樹、資料表、日誌、圖表修改 LogContains 邏輯
FileDialogPage.cs開啟檔案對話框操作路徑輸入 fallback
MessageBoxPage.cs訊息對話框驗證與關閉TC06、TC07
App.configEXE 路徑、逾時、截圖設定ElementTimeout
TestHooks.csApp 啟動/關閉、報告生命週期BeforeScenario 邏輯

2.3 修改 Feature 步驟

Feature 檔路徑：

Automation_testcase/Project_FlaUIBDD/Testcase_demo2_desktop_FlaUI_BDD/Features/Demo2Desktop.feature

範例：在 TC03 新增 Then 斷言

@Functional @RawData Scenario: TC03 - RawData 參數表 Given 測試資料已就緒 And 應用程式已啟動 When 我點擊工具列「RawData」 Then 資料表應可見 And 日誌區應包含「RawData」 And 主視窗標題應為「Semi Inspection Desktop」 ← 新增步驟

新增步驟後，確認 Demo2DesktopSteps.cs 已有對應綁定；若無，需新增 Step 方法。

2.4 新增 Gherkin 步驟綁定

在 Demo2DesktopSteps.cs 新增方法，regex 必須與 Feature 步驟完全匹配：

[Then(@"主視窗標題應為「(.*)」")] public void ThenMainWindowTitleShouldBe(string expectedTitle) { var actual = _mainWindowPage.GetWindowTitle(); ClassicAssert.AreEqual(expectedTitle, actual); }
原則：Steps 只做編排與 Assert；FlaUI 找元素、點擊等操作放在 PageObject 方法中。

2.5 新增 TC 的規則

  • 一個 Scenario = 一個 TC，編號連續（TC11、TC12…）
  • 標題格式：TCxx - 簡短描述
  • 標籤：@Functional 或 @Negative + 領域標籤
  • 優先重用既有 Given/When/Then 步驟
  • 新步驟先在 TPS.md 定稿，再實作 Step 與 PageObject
  • 同步更新 TEST_PLAN.md 與 SemiInspection_10_TestCases.md

2.6 使用 FlaUI Inspector 確認控制項

修改 UI 相關 PageObject 前，建議用 Inspector 確認 AutomationId：

  1. 雙擊 Automation_testcase/Project_FlaUIBDD/Testcase_demo2_desktop_FlaUI_BDD/開啟Inspector.bat
  2. 選擇行程 SemiInspectionDesktop
  3. 檢查控制項的 Name、AutomationId、ControlType
  4. 更新 PageObject 與 SemiInspection_10_TestCases.md

若 Inspector 不存在，執行 tools/install-flauinspect.ps1 安裝。

2.7 修改 App.config

鍵說明修改時機
ApplicationPath被測 EXE 路徑（相對 APP_ROOT）建置輸出路徑變更
RecipeDataDirectoryRecipe_data 資料夾測試資料位置變更
SampleRecipe預設樣本 JSON更換測試資料檔名
DefaultTimeout / ElementTimeout等待逾時（毫秒）UI 載入變慢
TakeScreenshotOnClick每步操作截圖報告需更詳細截圖
TakeScreenshotOnFailure失敗時截圖除錯需求

環境變數可由 setup_env.bat 覆寫，優先於 App.config。

2.8 修改後驗證

cd SQA_Inspetion_App build_semi.bat 執行單一測試.bat TC03

或手動：

cd Automation_testcase\Project_FlaUIBDD\Testcase_demo2_desktop_FlaUI_BDD dotnet build -c Release dotnet test -c Release --filter "Name~TC03"
