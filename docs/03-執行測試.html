3.1 前置需求

  • Windows 10/11
  • .NET SDK 8.0+（專案目標 net8.0-windows）
  • Python 3.x（僅 Web 控制台需要）
  • MSBuild .NET 3.5（建置被測 WinForms App）

被測程式與測試資料：

SemiInspectionDesktop\bin\Debug\SemiInspectionDesktop.exe Recipe_data\InspectionRecipe_Sample.json

3.2 方式 A：批次檔一鍵執行（建議）

批次檔用途
run_tests.bat建置被測 App + BDD 專案，執行全部 10 個 TC
執行單一測試.bat TC01建置後只跑指定 TC（TC01–TC10）
build_semi.bat僅建置被測 SemiInspectionDesktop.exe
run_semi.bat手動啟動被測 App（不含測試）
啟動測試平台.bat開啟 Web 控制台 http://localhost:6690/
開啟測試報告.bat開啟最新 TestResultReport.html

執行全部測試

cd SQA_Inspetion_App run_tests.bat

內部流程：setup_env.bat → build_semi.bat → dotnet build -c Release → dotnet test -c Release

執行單一 TC

執行單一測試.bat TC01 執行單一測試.bat TC07

3.3 方式 B：手動命令列

完整流程

cd SQA_Inspetion_App build_semi.bat cd Automation_testcase\Project_FlaUIBDD\Testcase_demo2_desktop_FlaUI_BDD dotnet build -c Release dotnet test -c Release --no-build

單一 TC

dotnet test -c Release --filter "Name~TC01"

多個 TC

dotnet test -c Release --filter "Name~TC01|Name~TC03|Name~TC05"

詳細 log + JUnit 報告

dotnet test -c Release ^ --logger "junit;LogFilePath=reports\junit-results.xml" ^ --logger "console;verbosity=detailed"

3.4 方式 C：Web 控制台（圖形化）

  1. 雙擊 啟動測試平台.bat
  2. 瀏覽器自動開啟 http://localhost:6690/
  3. 在「Features」分頁勾選 TC01–TC10
  4. 點「執行已勾選的測試」
  5. 在「執行進度」看即時 log
  6. 在「測試結果」點 HTML 報告連結
Web 控制台後端為 Automation_testcase/Project_FlaUIBDD/web_dashboard/server.py，會自動呼叫 dotnet test 並同步報告到 FlaUI 專案 reports/ 目錄。

3.5 環境變數（setup_env.bat）

run_tests.bat 與 啟動測試平台.bat 會自動載入：

變數預設值說明
APP_ROOT專案根目錄ConfigHelper 向上尋找含 config.json 的目錄
ApplicationPath{APP_ROOT}\SemiInspectionDesktop\bin\Debug\SemiInspectionDesktop.exe被測 EXE
RecipeDataDirectory{APP_ROOT}\Recipe_data測試資料目錄
ProcessNameSemiInspectionDesktop程序名稱
ApplicationTitleSemi Inspection Desktop主視窗標題

3.6 測試執行生命週期

Hook時機行為
BeforeTestRun全部測試開始前初始化 ExtentReports、TestResultReportBuilder
BeforeScenario每個 TC 開始Launch/Attach App、建立 Page Objects
BeforeStep / AfterStep每個 Gherkin 步驟記錄步驟、截圖、Pass/Fail
AfterScenario每個 TC 結束Scenario 完成截圖、關閉 App
AfterTestRun全部測試結束Flush HTML 報告

每個 Scenario 獨立 Launch/Kill 被測程序，確保 TC 之間互不干擾。

  2. 若有失敗，參考 第 4 章：修復測試
