4.1 修復流程總覽

  1. 確認被測 App 已建置 — build_semi.bat
  2. 縮小範圍 — 只跑失敗的 TC：執行單一測試.bat TCxx
  3. 讀取報告 — 開啟測試報告.bat，找到失敗步驟與 error message
  4. 查看截圖 — bin/Release/net8.0-windows/Screenshots/
  5. 分類錯誤 — 依下表判斷修改位置
  6. Inspector 除錯 — 確認 AutomationId（UI 相關失敗）
  7. 修改程式碼 — PageObject / App.config / Steps
  8. 重新驗證 — 單一 TC 通過後再跑全部

4.2 常見失敗類型與修復方式

失敗類型症狀檢查點修改位置
找不到 EXE FileNotFoundException、ApplicationPath 錯誤 是否執行 build_semi.bat App.config、setup_env.bat
UIA Timeout 找不到控制項、WaitUntilEnabled 逾時 Inspector 確認 AutomationId PageObjects、ElementTimeout
日誌 Assert 失敗 the log should contain "xxx" failed 實際 log 內容 vs 預期字串 WorkspacePage.LogContains、Then 步驟
TC01 檔案對話框 選檔失敗、路徑輸入無效 IME 環境、modal 對話框 FileDialogPage（1148、剪貼簿 fallback）
TC07–TC09 MessageBox 對話框未關閉或找不到 對話框標題/按鈕文字 MessageBoxPage
TC08/TC09 主視窗 App crash 或主視窗消失 被測 App 是否異常退出 MainWindowPage.IsMainWindowVisible
Step 綁定失敗 BindingException、找不到步驟定義 Feature 步驟 vs Steps regex Inspection_AppSteps.cs
測試資料缺失 Recipe_data 找不到樣本檔 Recipe_data 目錄內容 TestDataHelper、App.config

4.3 使用 FlaUI Inspector 除錯

  1. 雙擊 Automation_testcase\Project_FlaUIBDD\Testcase_Inspection_App_FlaUI_BDD\開啟Inspector.bat
  2. 若 Inspector 不存在，執行 tools\install-flauinspect.ps1
  3. 在 Inspector 選擇 SemiInspectionDesktop 行程
  4. 點選失敗步驟涉及的控制項，記錄：
    • AutomationId（如 btnImportRecipe、treeFiles）
    • Name / ControlType
    • 是否 Enabled / Visible
  5. 對照 Automation_testcase\Test_cases\SemiInspection_10_TestCases.md 與 PageObject 程式碼

4.4 詳細 log 重跑

cd Automation_testcase\Project_FlaUIBDD\Testcase_Inspection_App_FlaUI_BDD dotnet test -c Release --filter "Name~TC01" --logger "console;verbosity=detailed"

詳細 log 會顯示每個步驟的執行順序與 Assert 失敗的完整 stack trace。

4.5 修復 PageObject 範例

控制項 AutomationId 變更

若 Inspector 顯示 Import 按鈕的 AutomationId 從 btnImportRecipe 改為 btnImport：

// MainWindowPage.cs — ToolbarMap ["Import Recipe"] = ("btnImport", VirtualKeyShort.KEY_I),

修改檔案：MainWindowPage.cs

延長逾時

<!-- App.config --> <add key="ElementTimeout" value="60000" />

日誌關鍵字不匹配

報告顯示實際 log 為「Import completed」，但 Then 步驟預期 the log should contain "Import Recipe"：

  • 方案 A：修改 Feature/TPS 的 Then 步驟為實際 log 內容
  • 方案 B：修改被測 App 的 log 輸出（若為 App bug）

4.6 已知限制與注意事項

  • FileDialog + IME：繁體中文輸入法環境下，FileDialogPage 使用剪貼簿 + 鍵盤 fallback
  • 每 Scenario 獨立程序：TC 之間會 Kill/Launch App，確保狀態乾淨
  • 執行前關閉其他 modal：避免檔案對話框被其他視窗遮擋
  • Release 組態：正式執行使用 -c Release，報告產生在 Release 輸出目錄

4.7 修復完成驗證

cd SQA_Inspetion_App build_semi.bat 執行單一測試.bat TCxx REM 單一 TC 通過後，跑全部回歸 run_tests.bat

確認 開啟測試報告.bat 中該 TC 為 PASSED，且步驟截圖正常。報告解讀見 第 5 章。
